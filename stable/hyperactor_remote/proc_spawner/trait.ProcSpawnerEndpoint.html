fn spawn_proc_uid_with_ready( &self, cx: &impl Actor, uid: Uid, ready: OncePortHandle<()>, ) -> Result<ActorRef<SpawnActor>>
where Self: Clone + Send + 'static, for<'a> &'a Self: Endpoint<SpawnProc>,

Spawn a proc and report when it becomes reachable.

The returned ActorRef names the proc’s actor-spawn endpoint, but the proc has to launch and join before that endpoint can route — messages sent earlier are dropped. A bare readiness signal is posted to ready once the proc has joined, so callers that need to use the endpoint can wait for that signal instead of guessing with a delay. The endpoint address is already known from the synchronously returned ActorRef, so ready carries no payload.
