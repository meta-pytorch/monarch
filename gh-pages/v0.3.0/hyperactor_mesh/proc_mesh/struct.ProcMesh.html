A ProcMesh maintains a mesh of procs whose lifecycles are managed by an allocator.

pub async fn spawn<A: RemoteSpawn, C: Actor>( &self, cx: &C, actor_name: &str, params: &A::Params, ) -> Result<RootActorMesh<'_, A>, Error>
where A::Params: RemoteMessage, C::A: Handler<MeshFailure>,

Spawn an ActorMesh by launching the same actor type on all agents, using the same parameters instance for every actor.

  • actor_name: Name for all spawned actors.
  • params: Reference to the parameter struct, reused for all actors.

Bounds:

  • A: Actor — we actually spawn this type on each agent.
  • A: Referable — we return a RootActorMesh<'_, A> that contains ActorRef<A>s; those exist only for A: Referable.
  • A::Params: RemoteMessage — params must be serializable to cross proc boundaries when launching each actor.
