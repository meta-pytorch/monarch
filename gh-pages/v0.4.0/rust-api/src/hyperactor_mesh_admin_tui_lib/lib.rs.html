9//! Interactive TUI client for the Monarch mesh admin HTTP API.
10//!
11//! Displays the mesh topology as a navigable tree by walking `GET
12//! /v1/{reference}` endpoints. Selecting any node shows contextual
13//! details on the right pane, including actor flight recorder events
14//! when an actor is selected.
15//!
16//! Binaries construct a [`TuiConfig`] (after resolving any
17//! `mast_conda:///` handles themselves) and call [`run`].
18//!
19//! # Design Pillars (Algebraic)
20//!
21//! This TUI is intentionally structured around three algebraic
22//! invariants to make behavior correct by construction:
23//!
24//! 1. **TUI-1 (join-semilattice):** All fetch results merge via
25//!    `FetchState::join`, guaranteeing commutativity, associativity,
26//!    and idempotence under retries and reordering.
27//! 2. **TUI-2 (cursor-bounds):** Selection is managed by `Cursor`, which
28//!    enforces the invariant `pos < len` (or `pos == 0` when empty).
29//! 3. **TUI-3 (tree-recursion):** The mesh topology is stored
30//!    as an explicit tree (`TreeNode { children }`) and rendered via
31//!    a pure projection (`flatten_tree`), avoiding ad-hoc list
32//!    surgery.
33//!
34//! Additional invariants enforced throughout the code:
35//!
36//! - **TUI-4 (failed-always-visible):** Failed nodes are always
37//!   visible regardless of the `show_stopped` toggle.
38//! - **TUI-5 (single-fetch-path):** All cache writes go through
39//!   `fetch_with_join` (no direct inserts).
40//! - **TUI-6 (refresh-staleness):** `FetchState::Ready` with
41//!   `generation < refresh_gen` is refetched; errors always retry.
42//! - **TUI-7 (synthetic-root):** The root node is synthetic and
43//!   always expanded; only its children are rendered at depth 0.
44//! - **TUI-8 (cycle-safety):** Tree building rejects only true
45//!   cycles (nodes that appear in their own ancestor path).
46//! - **TUI-9 (depth-cap):** Recursion is bounded by
47//!   `MAX_TREE_DEPTH` (Root→Host→Proc→Actor→ChildActor).
48//! - **TUI-10 (fold-traversal):** Walks over `TreeNode` use fold
49//!   abstractions, not bespoke recursion.
50//! - **TUI-11 (selection-semantics):** Cursor restoration prefers
51//!   `(reference, depth)` to disambiguate; falls back to
52//!   reference-only if depth changes.
53//! - **TUI-12 (serial-fetches):** HTTP fetches are scheduled
54//!   serially; join semantics handle retries and reordering.
55//! - **TUI-13 (stopped-detection):** `is_stopped_node` matches
56//!   `Actor` variants whose `actor_status` starts with `"stopped:"`
57//!   or `"failed:"`. All other variants return false.
58//! - **TUI-14 (stopped-dual-source):** `TreeNode.stopped` is true
59//!   if the cached payload is stopped OR the child ref appears in
60//!   the parent proc's `stopped_children` list.
61//! - **TUI-15 (filter-order):** When both filters are off, system
62//!   membership is checked first.
63//! - **TUI-16 (placeholder-equivalence):** `placeholder_stopped` is
64//!   identical to `placeholder` except `stopped: true`.
65//! - **TUI-17 (failure-propagation):** A node's `failed` flag is
66//!   `is_failed_node(payload) || children.any(failed)`. Host/root
67//!   nodes are failed only when a descendant is.
68//! - **TUI-18 (collapsed-failure-carry):** When collapsed, prior
69//!   `failed` state is carried forward via `failed_keys`. Expanded
70//!   nodes always recompute from live children.
71//! - **TUI-19 (system-dual-source):** `TreeNode.is_system` is true
72//!   if `is_system: true` in payload OR child ref in parent's
73//!   `system_children`. Style precedence: selected > stopped >
74//!   system > node-type.
75//! - **TUI-20 (proc-accounting):** Displayed total is `num_actors +
76//!   stopped_children.len()`. `"(max retained)"` appears iff cap
77//!   reached.
78//!
79//! TLS and transport invariants:
80//!
81//! - **TUI-T1 (tls-auto-detect):** `client::build_client` probes
82//!   for TLS material in priority order: explicit CLI paths →
83//!   `try_tls_pem_bundle` → plain HTTP fallback.
84//! - **TUI-T2 (prebuilt-client):** `App::new` receives a pre-built
85//!   `reqwest::Client` and `base_url` (including scheme). TLS
86//!   configuration is external to the app state.
87//! - **TUI-T3 (scheme-inclusive-url):** `base_url` always starts
88//!   with `http://` or `https://`; bare `host:port` is resolved to
89//!   a scheme during client construction, never stored schemeless.
90//! - **TUI-21 (job-overlay-coherence):** `active_job.is_some() ↔
91//!   overlay.is_some()`. Enforced by `set_job`/`dismiss_job` —
92//!   all creation goes through `set_job` (which calls
93//!   `rebuild_overlay`) and all teardown through `dismiss_job`
94//!   (which clears both fields). Makes it structurally impossible
95//!   to have an orphaned overlay or a running job with no display
96//!   surface.
97//!
98//! Py-spy overlay invariants:
99//!
100//! - **PY-1 (fresh-trace):** Every `p` press issues a new HTTP
101//!   fetch; no cached py-spy result is ever reused.
102//! - **PY-2 (overlay-ownership):** A py-spy result may only populate
103//!   a still-valid active py-spy overlay. Stale results are
104//!   invalidated structurally: `active_job` carries the receiver
105//!   inside the `PySpy` variant, so replacing or clearing `active_job`
106//!   drops the receiver and cancels any in-flight fetch. TUI-21
107//!   guarantees `active_job` is always `None` when `overlay` is `None`.
108//! - **PY-3 (replacement):** Opening or refreshing py-spy replaces
109//!   prior overlay content and resets scroll to zero on result
110//!   arrival.
111//! - **PY-4 (selection-totality):** `p` is a no-op on Root/Host;
112//!   targets the proc ref directly on Proc; targets the owning proc
113//!   via `detail.parent` on Actor.
114//! - **PY-5 (overlay-isolation):** Diagnostics and py-spy overlays
115//!   must not write into each other's display surface. Enforced by
116//!   `set_job`: `RunDiagnostics` calls `set_job` with the
117//!   `Diagnostics` variant (dropping any live `PySpy` receiver);
118//!   Esc calls `dismiss_job`; `recv_active_job` fires only for the
119//!   variant currently stored.
120//!
121//! Config overlay invariants:
122//!
123//! - **CFG-1 (fresh-snapshot):** Every `C` press issues a new HTTP
124//!   fetch; no cached config result is ever reused.
125//! - **CFG-2 (overlay-ownership):** A config result may only populate
126//!   a still-valid active config overlay. Stale results are
127//!   invalidated structurally: `active_job` carries the receiver
128//!   inside the `Config` variant, so replacing or clearing `active_job`
129//!   drops the receiver and cancels any in-flight fetch. TUI-21
130//!   guarantees `active_job` is always `None` when `overlay` is `None`.
131//! - **CFG-3 (replacement):** Opening or refreshing config replaces
132//!   prior overlay content and resets scroll to zero on result
133//!   arrival.
134//! - **CFG-4 (selection-totality):** `C` targets the proc ref directly
135//!   on Proc (worker or service); targets the owning proc via
136//!   `detail.parent` on Actor; no-op on Root/Host. Backend routes to
137//!   ProcAgent or HostAgent per proc type (same as PY-4).
138//! - **CFG-5 (overlay-isolation):** Config, Diagnostics, and PySpy
139//!   overlays must not write into each other's display surface.
140//!   Enforced by `set_job`: each variant drops any prior receiver;
141//!   `recv_active_job` fires only for the variant currently stored.
142//!
143//! Laziness + recursion benefits:
144//! - **Lazy expansion**: proc/actor children are placeholders until
145//!   expanded, keeping refresh costs bounded and scaling work to what
146//!   the user explores.
147//! - **Structural recursion**: tree operations are defined by
148//!   recursion/folds over the explicit tree, avoiding brittle
149//!   index-based manipulation and making the view a pure projection.
150//!
151//! ```bash
152//! # Terminal 1: Run dining philosophers (or any hyperactor application)
153//! buck2 run fbcode//monarch/hyperactor_mesh:hyperactor_mesh_example_dining_philosophers
154//!
155//! # Terminal 2: Run this TUI (use the port printed by the application)
156//! buck2 run fbcode//monarch/hyperactor_mesh_admin_tui:hyperactor_mesh_admin_tui -- --addr 127.0.0.1:XXXXX
157//! ```

215/// Configuration for the mesh admin TUI.
216///
217/// Binaries translate their own CLI args into this struct.
218/// The `addr` field must be a resolved address (`host:port` or
219/// `https://host:port`), not a MAST handle. MAST resolution is
220/// the binary's responsibility.
221pub struct TuiConfig {
222    pub addr: String,
223    pub refresh_ms: u64,
224    pub theme: ThemeName,
225    pub lang: LangName,
226    pub tls_ca: Option<String>,
227    pub tls_cert: Option<String>,
228    pub tls_key: Option<String>,
229    pub diagnose: bool,
230}

234/// Put the terminal into "TUI mode".
235fn setup_terminal() -> io::Result<Terminal<CrosstermBackend<io::Stdout>>> {
236    enable_raw_mode()?;
237    let mut stdout = io::stdout();
238    stdout.execute(EnterAlternateScreen)?;
239    let backend = CrosstermBackend::new(stdout);
240    let mut terminal = Terminal::new(backend)?;
241    terminal.clear()?;
242    Ok(terminal)
243}
244
245/// Restore the terminal back to normal "shell mode".
246fn restore_terminal(terminal: &mut Terminal<CrosstermBackend<io::Stdout>>) -> io::Result<()> {
247    disable_raw_mode()?;
248    terminal.backend_mut().execute(LeaveAlternateScreen)?;
249    terminal.show_cursor()?;
250    Ok(())
251}

253async fn run_diagnose(client: reqwest::Client, base_url: String) -> io::Result<()> {
254    use crate::diagnostics::DiagSummary;
255    use crate::diagnostics::run_diagnostics;
256
257    const GLOBAL_TIMEOUT_SECS: u64 = 120;
258
259    let mut rx = run_diagnostics(client, base_url);
260    let mut results = Vec::new();
261
262    let timed_out = tokio::time::timeout(Duration::from_secs(GLOBAL_TIMEOUT_SECS), async {
263        while let Some(r) = rx.recv().await {
264            results.push(r);
265        }
266    })
267    .await
268    .is_err();
269
270    let s = DiagSummary::from_results(&results);
271    let healthy = s.passed == s.total && !timed_out;
272
273    let report = serde_json::json!({
274        "checks": results,
275        "timed_out": timed_out,
276        "summary": {
277            "total": s.total,
278            "passed": s.passed,
279            "failed": s.total - s.passed,
280            "admin_infra_passed": s.admin_passed,
281            "admin_infra_total": s.admin_total,
282            "mesh_passed": s.mesh_passed,
283            "mesh_total": s.mesh_total,
284            "healthy": healthy,
285        }
286    });
287
288    println!(
289        "{}",
290        serde_json::to_string_pretty(&report).unwrap_or_else(|e| format!("{{\"error\": \"{e}\"}}"))
291    );
292
293    if !healthy {
294        std::process::exit(1);
295    }
296    Ok(())
297}

299/// Run the mesh admin TUI. Does not return until the user exits
300/// or diagnostics complete.
301pub async fn run(config: TuiConfig) -> io::Result<()> {
302    let (base_url, client) = client::build_client(&config);
303
304    if config.diagnose {
305        return run_diagnose(client, base_url).await;
306    }
307
308    if !io::stdout().is_terminal() {
309        eprintln!("This TUI requires a real terminal.");
310        return Ok(());
311    }
312
313    let mut app = App::new(base_url, client, config.theme, config.lang);
314    let spinner = ProgressBar::new_spinner();
315    spinner.set_style(
316        ProgressStyle::default_spinner()
317            .template("{spinner:.cyan} {msg}")
318            .expect("valid template"),
319    );
320    spinner.set_message(format!("mesh-admin — Connecting to {} ...", app.base_url));
321    spinner.enable_steady_tick(Duration::from_millis(80));
322
323    let splash_start = tokio::time::Instant::now();
324    app.refresh().await;
325    let elapsed = splash_start.elapsed();
326    let min_splash = Duration::from_secs(2);
327    if elapsed < min_splash {
328        tokio::time::sleep(min_splash - elapsed).await;
329    }
330
331    spinner.finish_and_clear();
332
333    let mut terminal = setup_terminal()?;
334    let result = run_app(&mut terminal, config.refresh_ms, app).await;
335    restore_terminal(&mut terminal)?;
336    result
337}
